# 💰 Finance Dashboard UI

A modern, responsive, and interactive **Finance Dashboard** built using React.
This project focuses on clean UI design, structured component architecture, and effective state management to simulate real-world financial tracking.

## 📌 Project Overview

This dashboard allows users to:

* View overall financial summary
* Explore and manage transactions
* Understand spending patterns through visualizations
* Interact with a role-based UI (Viewer/Admin)

The goal of this project is to demonstrate **frontend development skills, UI/UX design thinking, and scalable code structure**.

---

## ✨ Features

### 📊 Dashboard Overview

* Summary cards:

  * Total Balance
  * Total Income
  * Total Expenses
* Time-based visualization (Balance trend)
* Category-based visualization (Spending breakdown)

---

### 💳 Transactions Section

* Display transactions with:

  * Date
  * Amount
  * Category
  * Type (Income/Expense)

* Functionalities:

  * 🔍 Search
  * 🎯 Filter (Income/Expense)
  * ↕️ Sorting (Date/Amount)

* Admin-only features:

  * ➕ Add transaction
  * ✏️ Edit transaction
  * ❌ Delete transaction

---

### 🔐 Role-Based UI (Simulated)

* Viewer → Can only view data
* Admin → Can modify transactions

Role switching implemented via dropdown for demonstration.

---

### 📈 Insights Section

* Highest spending category
* Monthly comparison
* Total savings (Income - Expenses)
* Smart observations based on data

---

### 🎨 UI/UX Highlights

* Clean and modern fintech-inspired design
* Fully responsive (mobile, tablet, desktop)
* Smooth transitions and hover effects
* User-friendly layout and navigation
* Graceful handling of empty states

---

## ⚙️ Tech Stack

* **Frontend:** React (Functional Components + Hooks)
* **Styling:** Tailwind CSS
* **Charts:** Recharts / Chart.js
* **State Management:** Context API / Zustand
* **Icons:** Lucide / Heroicons

---

## 🧠 State Management

The application manages:

* Transactions data
* Filters and search state
* Selected user role
* Chart data

State is structured to ensure **scalability, readability, and separation of concerns**.
